Goal: Add an “Add to Cart” system and “Bundles for Popular Moments” on collection pages. Bundles are curated sets of 3 products for specific moments (dorm move-in, first apartment styling, wedding guest-ready, etc). Users should be able to add single items to cart, add a full bundle to cart, or “Buy Bundle Now” which creates a Stripe Checkout session containing all bundle items at once. We should also show a bundle discount and the percent savings vs buying items individually.

Constraints:
	•	Keep it simple and reliable. Minimal new dependencies.
	•	Cart should persist across page navigation and refresh using localStorage.
	•	The bundle discount is applied in Stripe so the checkout total matches the site.
	•	We need analytics events (GA4) for bundle impressions, bundle add to cart, and bundle checkout starts.

Deliverables:
	1.	Data model for cart and bundles
	2.	UI changes on collection pages
	3.	Cart UI (slide-over or modal) with quantity editing, remove, subtotal, and checkout
	4.	Stripe changes to support multi-item checkout (single product checkout still works)
	5.	GA4 tracking for new events
	6.	Testing checklist

Implementation details:

A. Cart
	•	Create a cart store using React Context or a lightweight custom hook.
	•	Cart item schema:
	•	product_id: string
	•	variant_id?: string (if variants exist)
	•	name: string
	•	price: number (unit price shown on site)
	•	image_url: string
	•	quantity: number
	•	collection_slug?: string
	•	source: 'single' | 'bundle'
	•	bundle_id?: string
	•	Persist cart to localStorage under key ecoshopguide_cart_v1.
	•	Provide functions: addItem, removeItem, updateQty, clearCart, addBundle(bundle).

B. Bundles
	•	Add a bundles.ts config file that maps collection_slug to an array of bundle definitions.
	•	Bundle schema:
	•	bundle_id: string (unique)
	•	title: string (moment headline)
	•	description: string (1 sentence)
	•	collection_slug: string
	•	product_ids: string[] length 3
	•	discount_type: 'percent' | 'fixed'
	•	discount_value: number (ex: 10 for 10% off)
	•	Bundle products must be pulled from existing product catalog data. If a product id is missing, the bundle should gracefully not render.

E. Stripe checkout changes
We currently have POST /api/checkout expecting a single product.
Update to support both:
	1.	Single item checkout: existing behavior remains
	2.	Cart checkout: accepts an array of items and optional bundle discount logic

API proposal:
	•	POST /api/checkout
	•	Body can be:
	•	{ mode: 'single', product_id, variant_id?, quantity }
	•	{ mode: 'cart', items: [{ product_id, variant_id?, quantity, bundle_id? }], bundle_discounts?: [{ bundle_id, discount_type, discount_value }] }

Server responsibilities:
	•	Validate product ids and pricing from server-side canonical product data (do not trust client prices).
	•	Build line_items for Stripe Checkout session with correct quantities.
	•	Apply bundle discount:
	•	Option 1: use Stripe Coupons (percent off) and apply if cart contains a full bundle set.
	•	Option 2: use Stripe Checkout “discounts” with a coupon created in Stripe Dashboard (like BUNDLE10) and apply once per order if at least one bundle is present.
Start with simple approach:
	•	If cart contains any bundle items, apply 10% coupon to entire order.
	•	Later we can refine to bundle-specific discounting.

Return { url } and client redirects.

Also add Stripe success and cancel URLs to return to site and clear cart on success page load.

	•	cart_view when cart drawer opens with params:

	•	cart_checkout_start when user clicks Checkout with params:

	•	bundle_buy_now_click when user clicks Buy Bundle Now with params:
	•	bundle_id, collection_slug, bundle_value
	•	add_to_cart for single product add with params:
	•	product_id, collection_slug, price
	•	remove_from_cart similarly

Also implement a helper gaTrack(eventName, params) that:

	•	logs to console only in development mode with prefix [GA4] and JSON-safe payloads (avoid circular JSON errors)
